    This pipeline is currently not loading on a schedule.

    More info: MFA changes

    Snowflake is blocking password based authentication for your service user. You will need to customize the service user by setting the TYPE property to LEGACY_SERVICE.

    1) Log into your Snowflake account. You must have Account Admin privileges to alter the service user.

    2) Identify the Snowflake service user, data warehouse, and database used for your Precog data destination.

    3) Create a new SQL worksheet.

    4) Add the following line, replacing precog with the Snowflake service user name. ALTER USER precog SET TYPE=LEGACY_SERVICE; Run the script.

    5) Verify that the TYPE has been updated. In the left hand navigation click on the Admin icon. Select Users & Roles and then the service user which was modified. In the About section the User Type should be LEGACY_SERVICE.

    6) In your Precog Pipeline click on the “Reload datasets”. Select your datasets, and then click on “Reload selected datasets”. In the dialog box, click on the “Reload only” option to connect to your Snowflake destination.
